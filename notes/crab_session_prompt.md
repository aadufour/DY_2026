**2. `fileset.json`** — per-config-dir, auto-generated by `spritz-fileset`
from `samples.json`, then manually patched to duplicate each dataset's file
list across 55 subsample keys (`sm`, `w1_<op>`, `wm1_<op>` for 27
operators) — the physical NanoAOD files are the same for every subsample,
only the *weight column selected at analysis time* differs. Patch pattern
(already used for baseline, from spritz.md):
```python
import json
path = "data/fileset.json"
with open(path) as f: s = json.load(f)
MLL_BINS = ["50_120","120_200","200_400","400_600","600_800","800_1000","1000_3000"]
OPERATORS = ["cHDD","cHWB","cbWRe","cbBRe","cHj1","cHQ1","cHj3","cHQ3",
             "cHu","cHd","cHbq","cHl1","cHl3","cHe","cll1","clj1","clj3",
             "cQl1","cQl3","ceu","ced","cbe","cje","cQe","clu","cld","cbl"]
for b in MLL_BINS:
    base = f"DYSMEFTsim_LO_mll_{b}"     # <-- will need a propcorr-equivalent base name
    parent = s[base].copy()
    s[f"{base}_sm"] = parent.copy()
    for op in OPERATORS:
        s[f"{base}_w1_{op}"]  = parent.copy()
        s[f"{base}_wm1_{op}"] = parent.copy()
with open(path, "w") as f: json.dump(s, f, indent=2)
```
This needs to run again (adapted for whatever propcorr dataset base name we
pick) after `spritz-fileset` regenerates/updates `fileset.json` for the new
samples.
